Introduction to the Pattern Library

A pattern library helps designers and developers maintain a consistent appearance throughout a web project’s life. By breaking apart the project into smaller components and stylistic rules, a baseline is created for building new components and pages that keep the same look and feel as the existing parts of the project.

Pattern libraries also provide useful reference material and documentation.

About the Library

At the top of this page are three tabs: Style, Components, and Pages. The smallest, most basic aspects of design are found in the Style section, more complex design combinations — or components — are placed in the Components section, and finalized page templates are found in the Pages section.

Style Section

You are in the Style section now — the Introduction to be precise. The design of this page is automatically pulled from the style sheets used by the web project. The headers and paragraphs you are seeing adhere to the basic typographical and color rules defined in the CSS. To see more information about the typography, colors, forms, and other basic design elements for this site, use the links below the Style tab at the top of this page.

Components Section

Components are larger, reusable elements that build on the base styles. Examples of components would be a tabbed interface, a modal dialog box, or even a blog comment. Before creating new designs, you should always reference the existing components to see if something already exists to meet your needs. If you still must create a new component, be sure to add it to this pattern library when it’s done.

Pages Section

The pattern library is expected to grow along with the web project. When starting out, the Pages section will contain basic wireframes that can be tested in live browsers. As design decisions are made, the wireframes will be fleshed out with the styles and components as they are created. When the project is finished, the pages act as a reference for the various layouts that are possible and can provide guidance when creating new page templates.
