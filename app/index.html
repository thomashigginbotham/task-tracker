Built for Responsive Design

Unlike other popular frameworks, XPL was designed from the ground up with responsive design in mind. It wasn’t tacked on and it doesn’t require additional or alternate CSS to work.

It’s Semantic

No need to learn a lot of cryptic class names like col-xs-7 or medium-3 columns end. Write your HTML with semantic class names, such as site-header or sidebar, and use our short list of Sass mixins to control the layout.

Fluid or Fixed-width Gutters

The gutter between these two columns is fluid (10% of the total width). Resize your browser to see its width adjust relative to the browser.

The other demos on this page use fixed-width gutters (1.4 rem). They stay the same size regardless of the browser width.

Set your gutters to be a percentage or a fixed-width value and even add backgrounds to the columns. Try doing that in Bootstrap or Foundation without pulling your hair out.

Sprites for High DPI (Retina) Screens

XPL Has mixins for generating high DPI sprite images and CSS automatically. If you’re already familiar with using Compass for sprites, you’ll be right at home.

Take a look at app/stylesheets/modules/_sprites-2x.scss for the available functions and mixins.

Support for Jasmine/Karma Unit Testing

One of the benefits of building websites with a componentized approach is unit testing. XPL comes with a simple starter spec (test/sample-spec.js) that includes a single unit test for the included sample jQuery plugin (app/javascripts/lib/jquery-plugins/jquery.sampleplugin.js).

Use the grunt serve-test task to start up a server and run your unit tests as you make changes.
